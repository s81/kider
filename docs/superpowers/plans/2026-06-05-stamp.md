# Stamp Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `stamp()` builtin that leaves a filled turtle-shaped triangle at the turtle's current position and heading, enabling animation loops like `clearCanvas(); forward(x); stamp()`.

**Architecture:** Two tasks — (1) lang layer: new `stamp` Drawing variant, `mkStamp` constructor, renderer support, `stamp` builtin, `drawStamp` CanvasCommand, `drawUpTo` case; (2) blocks layer: one new `sprout_stamp` value block + compiler case + toolbox entry. No new AST node needed — `stamp()` is a plain `CallExpr`.

**Tech Stack:** TypeScript, Vitest (`bun test`), Blockly 10, Vite + React, HTML5 Canvas
