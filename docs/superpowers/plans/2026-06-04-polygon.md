# Polygon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `polygon(n, size)` as a filled Drawing primitive — a regular n-sided polygon centered on the turtle, filled semi-transparently and stroked at full opacity.

**Architecture:** Follows the exact same pipeline as `circle`, `rect`, `ellipse`, and `triangle`: Drawing variant → CanvasCommand → drawUpTo case → Blockly block. Geometry uses circumradius `R = size / (2 * sin(π/n))` with first vertex at top (−π/2 offset).

**Tech Stack:** TypeScript, Vitest, Blockly 10, pnpm monorepo (`@sprout/lang`, `@sprout/blocks`, `@sprout/ide`)

function roundCmd(c: CanvasCommand): CanvasCommand {
  if (c.kind === 'lineTo' || c.kind === 'moveTo') {
    return { kind: c.kind, x: roundNum(c.x), y: roundNum(c.y) };
  }
  if (c.kind === 'drawCircle')   return { ...c, x: roundNum(c.x), y: roundNum(c.y) };
  if (c.kind === 'drawRect')     return { ...c, x: roundNum(c.x), y: roundNum(c.y) };
  if (c.kind === 'drawEllipse')  return { ...c, x: roundNum(c.x), y: roundNum(c.y) };
  if (c.kind === 'drawTriangle') return { ...c, x: roundNum(c.x), y: roundNum(c.y) };
  if (c.kind === 'drawPolygon')  return { ...c, x: roundNum(c.x), y: roundNum(c.y) };
  return c;
}
