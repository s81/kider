# wait(seconds) — Implementation Plan
**Date:** 2026-06-09
**Spec:** docs/superpowers/specs/2026-06-09-wait-design.md

## Context

Add `wait(seconds)` builtin + animated playback to the Sprout IDE. The interpreter stays synchronous — `wait` returns a `WaitDrawing` value that flows through the renderer to become a `wait` CanvasCommand. The IDE then plays commands back with timing, pausing at each wait.

### Task 4 — stage-utils.ts: skip wait in drawUpTo + buildPlayback

File: `apps/ide/src/stage-utils.ts`

1. In `drawUpTo`, add `case 'wait': break;` — wait commands are timing markers only, not drawing ops.

2. Add `PlaybackSegment` type and `buildPlayback` function:

   ```ts
   export type PlaybackSegment =
     | { kind: 'draw'; upTo: number }
     | { kind: 'wait'; durationMs: number };

   export function buildPlayback(commands: CanvasCommand[]): PlaybackSegment[] {
     const segments: PlaybackSegment[] = [];
     let drawEnd = -1;
     for (let i = 0; i < commands.length; i++) {
       const cmd = commands[i];
       if (cmd.kind === 'wait') {
         if (drawEnd >= 0) segments.push({ kind: 'draw', upTo: drawEnd });
         segments.push({ kind: 'wait', durationMs: cmd.durationMs });
         drawEnd = -1;
       } else {
         drawEnd = i;
       }
     }
     if (drawEnd >= 0) segments.push({ kind: 'draw', upTo: drawEnd });
     return segments;
   }
   ```

### Task 5 — App.tsx: animated playback

File: `apps/ide/src/App.tsx`

1. Add `animTimeoutsRef = useRef<ReturnType<typeof setTimeout>[]>([])` to track active animation timeouts.

2. Add a `cancelAnimation()` helper that clears all timeouts in `animTimeoutsRef.current` and resets it.

3. Call `cancelAnimation()` at the top of `handleRun` (before any new execution) and also when the canvas is cleared/reset.

4. After the existing `setCommands(render(drawing))` line, add animated playback logic:

   ```ts
   const cmds = render(drawing);
   setCommands(cmds);
   cancelAnimation();

   const hasWait = cmds.some(c => c.kind === 'wait');
   if (!hasWait) {
     // instant draw — existing behavior
     drawToCanvas(cmds, cmds.length);
   } else {
     // animated playback
     const segments = buildPlayback(cmds);
     let cumIdx = -1;
     let segIdx = 0;

     function playNext() {
       if (segIdx >= segments.length) return;
       const seg = segments[segIdx++];
       if (seg.kind === 'draw') {
         cumIdx = seg.upTo;
         drawToCanvas(cmds, cumIdx + 1);
         playNext();
       } else {
         const tid = setTimeout(playNext, seg.durationMs);
         animTimeoutsRef.current.push(tid);
       }
     }
     playNext();
   }
   ```

   Note: `drawToCanvas` refers to whatever mechanism App.tsx uses to draw — either calling the canvas ref directly or through the existing command rendering path. Adapt to match how the app currently triggers `drawUpTo`.
