**Goal:** Add localStorage auto-save/restore and file export/import to the Sprout IDE so users never lose their work.

**Architecture:** A pure `storage.ts` module owns serialization logic (unit-tested); `App.tsx` wires auto-save via a `useEffect`, text-mode restore on mount, and blocks-mode restore in `onWorkspaceReady`; `BlockWorkspace.tsx` moves the `onWorkspaceReady` call to after the change listener is registered so restored blocks trigger `programText` updates.

**Tech Stack:** TypeScript, React, Blockly 10 (`Blockly.serialization`), localStorage, FileReader, Blob/URL

**Background:**

`SaveState` is a discriminated union. Blocks mode stores a JSON string (from `Blockly.serialization.workspaces.save`) in the `blocks` field plus the text representation. Text mode stores only the editor text. `parseSave` is the safety gate — always returns `null` rather than throwing on bad input.

```typescript
export type SaveState =
  | { mode: 'blocks'; blocks: string; text: string }
  | { mode: 'text';   text: string }

export function parseSave(json: string): SaveState | null {
  try {
    const obj = JSON.parse(json) as Record<string, unknown>;
    if (obj.mode === 'blocks') {
      if (typeof obj.blocks !== 'string' || typeof obj.text !== 'string') return null;
      return { mode: 'blocks', blocks: obj.blocks, text: obj.text };
    }
    if (obj.mode === 'text') {
      if (typeof obj.text !== 'string') return null;
      return { mode: 'text', text: obj.text };
    }
    return null;
  } catch {
    return null;
  }
}

export function buildBlocksSave(blocks: string, text: string): SaveState {
  return { mode: 'blocks', blocks, text };
}

export function buildTextSave(text: string): SaveState {
  return { mode: 'text', text };
}
```

### Task 2: Persistence wiring — auto-save and restore

**Files:**
- Modify: `apps/ide/src/BlockWorkspace.tsx`
- Modify: `apps/ide/src/App.tsx`

**Background:**

**BlockWorkspace.tsx timing fix:** Currently `onWorkspaceReady(ws)` is called before the change listener is added. When blocks-mode restore calls `Blockly.serialization.workspaces.load(state, ws)` inside `onWorkspaceReady`, the listener hasn't been registered yet, so `programText` doesn't update. Fix: move `onWorkspaceReady(ws)` to after `ws.addChangeListener(listener)`.

**Auto-save:** A `useEffect` watching `[sourceMode, programText, editorText]` writes to `localStorage.setItem('sprout_save', ...)` on every change. In blocks mode it captures fresh workspace state via `Blockly.serialization.workspaces.save(ws)`, JSON-stringifies it, and stores it via `buildBlocksSave`. In text mode it uses `buildTextSave(editorText)`.

**Restore on mount (text mode):** A `useEffect([], [])` reads the save on mount and restores text mode only (blocks mode can't restore here because `wsRef.current` is still null). Blocks restore happens in `onWorkspaceReady`.

**Restore in onWorkspaceReady (blocks mode):** Reads the save, checks `saved.mode === 'blocks'`, then calls `Blockly.serialization.workspaces.load(JSON.parse(saved.blocks), ws, { recordUndo: false })`. Wrapped in try/catch to silently ignore corrupt saves.

Read both files before editing.

### Task 3: File export/import + UI buttons

**Files:**
- Modify: `apps/ide/src/App.tsx`

**Background:**

Export creates a Blob from the current program text, triggers a download link click programmatically, then revokes the object URL. Import clicks a hidden `<input type="file">`, reads the selected file as text via `FileReader`, sets `editorText`, and switches to text mode. The hidden input's value is reset after each import so the same file can be re-imported.

Read `apps/ide/src/App.tsx` before editing.

- [ ] **Step 2: Add `handleExport` to `App.tsx`**

After `handleSwitchToEditor`, add:

```typescript
  function handleExport() {
    const text = sourceMode === 'blocks' ? programText : editorText;
    const blob = new Blob([text], { type: 'text/plain' });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = 'program.sprout';
    document.body.appendChild(a);
    a.click();
    document.body.removeChild(a);
    URL.revokeObjectURL(url);
  }
```

- [ ] **Step 3: Add `handleImport` and `handleFileChange` to `App.tsx`**

After `handleExport`, add:

```typescript
  function handleImport() {
    fileInputRef.current?.click();
  }

  function handleFileChange(e: React.ChangeEvent<HTMLInputElement>) {
    const file = e.target.files?.[0];
    if (!file) return;
    const reader = new FileReader();
    reader.onload = (ev) => {
      const text = ev.target?.result as string;
      setEditorText(text);
      setSourceMode('editor');
    };
    reader.readAsText(file);
    e.target.value = '';
  }
```
