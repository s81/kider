# List Operations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire five already-built list blocks into the IDE toolbox, and add six new list operations (`first`, `last`, `pop`, `concat`, `reverse`, and list-aware `contains`) across lang, blocks, compiler, and IDE.

**Architecture:** Each new builtin follows the established four-file pattern: runtime in `interpreter.ts`, Blockly block definition in `packages/blocks/src/definitions/values.ts`, compiler case in `packages/blocks/src/compiler.ts`, and toolbox entry in `apps/ide/src/BlockWorkspace.tsx`. Tests live in `packages/lang/tests/interpreter.test.ts` (runtime) and `packages/blocks/tests/compiler.test.ts` (block compiler). All ops are immutable and return new `SproutList` values.

**Tech Stack:** TypeScript, Vitest, Blockly 10, Bun (test runner)
