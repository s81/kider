# Else If Chains Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `else if` chain syntax to the Sprout language parser and serializer so kids can write multi-branch conditionals with a single `end`.

**Architecture:** `else if` is syntactic sugar — no AST change needed. The parser detects `else if` (ident `else` followed by ident `if`) and recursively parses the inner `if…end` as a single-statement `BlockExpr` for the outer's else slot, returning early to skip the duplicate `eatIdent('end')`. The serializer gains a `serializeIfChain` helper that flattens a nested `IfExpr`-in-else back to the `else if` keyword form. The interpreter is unchanged.

**Tech Stack:** TypeScript, Vitest (`bun test` from repo root), Bun

- [ ] **Step 3: Implement else-if detection in the parser**

In `packages/parser/src/parser.ts`, find the `if (this.checkIdent('else'))` block (around line 232). Replace:

```typescript
        if (this.checkIdent('else')) {
          this.advance();
          const elseStmts = this.parseBodyUntil(['end']);
          elseBlock = { kind: 'BlockExpr', body: elseStmts };
        }
        this.eatIdent('end');
        return { kind: 'IfExpr', cond, then: thenBlock, else: elseBlock } satisfies IfExpr;
```

With:

```typescript
        if (this.checkIdent('else')) {
          this.advance();
          if (this.checkIdent('if')) {
            const innerIf = this.parseExpr();
            elseBlock = { kind: 'BlockExpr', body: [{ kind: 'ExprStmt', expr: innerIf }] };
            return { kind: 'IfExpr', cond, then: thenBlock, else: elseBlock } satisfies IfExpr;
          }
          const elseStmts = this.parseBodyUntil(['end']);
          elseBlock = { kind: 'BlockExpr', body: elseStmts };
        }
        this.eatIdent('end');
        return { kind: 'IfExpr', cond, then: thenBlock, else: elseBlock } satisfies IfExpr;
```

The early `return` inside the `if (this.checkIdent('if'))` branch skips the outer `eatIdent('end')` — the recursively-parsed inner `if…end` already consumed that token.

```typescript
function serializeIfChain(expr: IfExpr, indentLevel: number, keyword: 'if' | 'else if'): string {
  const ind = indent(indentLevel);
  const condStr = serializeExpr(expr.cond, indentLevel);
  const thenStr = serializeBlock(expr.then, indentLevel + 1);
  if (expr.else === null) {
    return `${keyword} ${condStr} do\n${thenStr}\n${ind}end`;
  }
  const elseBody = expr.else.body;
  if (
    elseBody.length === 1 &&
    elseBody[0].kind === 'ExprStmt' &&
    elseBody[0].expr.kind === 'IfExpr'
  ) {
    const inner = serializeIfChain(elseBody[0].expr as IfExpr, indentLevel, 'else if');
    return `${keyword} ${condStr} do\n${thenStr}\n${ind}${inner}`;
  }
  const elseStr = serializeBlock(expr.else, indentLevel + 1);
  return `${keyword} ${condStr} do\n${thenStr}\n${ind}else\n${elseStr}\n${ind}end`;
}
```

Replace the `case 'IfExpr':` block (lines 85–93) with:
